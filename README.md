Optic Online Shop (Django + SQLite)

Минимальный интернет‑магазин оптической продукции (бинокли, телескопы, микроскопы, объективы, зеркала, линзы) на Django 5 и SQLite. Включает каталог, корзину на сессиях, оформление заказа только для авторизованных пользователей и админку Django.

Возможности
- Каталог: категории и товары
- Публичные страницы: список товаров с фильтром по категории, карточка товара
- Корзина: хранение в сессии (добавление, обновление, удаление)
- Оформление заказа: доступно только после входа (без оплат), статусы планируются
- Админка: управление категориями/товарами, просмотр заказов
- UI: Django Templates + Bootstrap (CDN), нормализация размеров изображений
  - Сетка карточек с одинаковой высотой превью
  - Страница входа по центру по горизонтали и выше середины по вертикали

Структура проекта
```
config/                 # Django-проект (settings/urls/asgi/wsgi)
catalog/                # Категории и товары (models, views, admin, urls)
cart/                   # Корзина на сессиях (views, urls, templates)
orders/                 # Чекаут (views, urls, templates)
templates/              # Базовые и экранные шаблоны (Bootstrap по CDN)
static/                 # Статика (css/app.css)
manage.py
README.md
```

Требования
- Python 3.11+ (рекомендуется 3.12)
- Windows PowerShell (команды ниже даны для PowerShell)

Быстрый старт (Windows PowerShell)
Запускайте из папки проекта, например `D:\My_Pet_Projects\OpticOnlineShop`.

```powershell
# 1) Виртуальное окружение и активация
py -m venv .venv
.\.venv\Scripts\Activate.ps1

# 2) Установить зависимости
python -m pip install --upgrade pip
pip install "Django>=5.0,<6"

# 3) Применить миграции
python manage.py migrate

# 4) Создать суперпользователя
python manage.py createsuperuser

# 5) Запуск сервера разработки
python manage.py runserver
```

Первые шаги после запуска
1) Откройте в браузере `http://127.0.0.1:8000/` — отобразится каталог (пока пустой).
2) Войдите как администратор:
   - Перейдите на `http://127.0.0.1:8000/accounts/login/` и введите логин/пароль суперпользователя.
   - Форма входа расположена по центру (горизонтально) и немного выше середины экрана.
3) Добавьте категории и товары через админку:
   - Зайдите на `http://127.0.0.1:8000/admin/`.
   - Раздел Categories → Add Category:
     - Примеры: Бинокли, Телескопы, Микроскопы, Объективы, Зеркала, Линзы.
     - Поле slug можно сгенерировать автоматически (заполнится из названия).
   - Раздел Products → Add Product:
     - Выберите категорию.
     - Укажите Name, Price (например 12990.00), Stock (наличие), поставьте галочку is_active.
     - В поле Image url укажите прямую ссылку на картинку (можно любой публичный URL с изображением товара).
     - Сохраните.
4) Вернитесь на `http://127.0.0.1:8000/`:
   - Каталог покажет товары с аккуратными превью одинаковой высоты.
   - Фильтр по категории доступен в левом сайдбаре (или верхнем блоке).
5) Проверьте корзину и оформление:
   - Откройте карточку товара → укажите количество → «В корзину».
   - Перейдите в `/cart/`: обновляйте количество (0 — удаляет позицию), удаляйте товары.
   - Нажмите «Оформить заказ» → заполните форму (ФИО, контакты, адрес, комментарий) → отправьте.
   - После отправки увидите страницу «Оформление прошло успешно» и кнопку «К товарам» (возврат в каталог). Корзина при этом очищается.

Основные URL
- `/` → список товаров
- `/catalog/` → список товаров (поддерживает фильтр `?category=<slug>`)
- `/catalog/<slug>/` → карточка товара (+ добавление в корзину)
- `/cart/` → корзина (обновление/удаление)
- `/orders/checkout/` → оформление заказа (требуется вход)
- `/accounts/login/` → вход (Django auth)
- `/accounts/logout/` → выход (POST-запрос, см. ниже)
- `/admin/` → админка Django

Аутентификация
- В Django 5 для `LogoutView` используется только POST. В навбаре уже сделана форма с CSRF, отправляющая POST на `/accounts/logout/`. Если будете делать собственную кнопку «Выйти», убедитесь, что это POST с CSRF.

Как работать через админку
1) Зайдите в `/admin/` под суперпользователем.
2) Создайте `Categories` (например: Бинокли, Телескопы, Микроскопы, Объективы, Зеркала, Линзы).
3) Создайте `Products`, укажите категорию, `price`, `stock`, `image_url` (прямая ссылка на изображение) и `is_active`.
4) Перейдите на `/` — товары появятся в каталоге.

Корзина и чекаут
- Корзина хранится в сессии пользователя, отдельной таблицы для корзины нет.
- В карточке товара можно добавить позицию с количеством (1–99).
- В `/cart/` можно обновлять количество (0 — удалить позицию) и переходить к оформлению.
- Оформление заказа доступно только для авторизованных; в текущем MVP — имитация (форма + очистка корзины + success‑страница). Реальные модели заказа появятся на следующем шаге.

Статика и изображения
- Bootstrap подключён с CDN.
- Локальные стили — `static/css/app.css`, подключены из `templates/base.html` через `{% load static %}` и `{% static %}`.
- Нормализация изображений:
  - `.product-thumb` — карточки списка (соотношение 4:3, `object-fit: cover`, единая высота);
  - `.product-hero` — страница товара (4:3, `object-fit: contain`, без обрезки).
  - Страница логина центрируется по горизонтали и смещена выше середины (см. стили контейнера на странице `templates/registration/login.html`).

Подсказки по разработке
- В `config/settings.py` уже подключены каталоги `templates/` и `static/` в корне.
- Для новых статических файлов используйте `{% load static %}` и путь через `{% static '...' %}`.
- Если захотите хранить изображения локально, переключитесь на `ImageField` и настройте `MEDIA_URL`/`MEDIA_ROOT` (в данный MVP не включено).

План развития
- Заказы: модели `Order` и `OrderItem`, форма (ФИО, телефон, email, комментарий), статусы.
- Админка: расширенные фильтры, поиск, вычисляемые поля (итоги и т.п.).
- Опционально: поиск по товарам, пагинация, SEO‑слаги, загрузка изображений, интеграции оплат и доставки.
